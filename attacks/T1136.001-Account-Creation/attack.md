## Overview

This module documents the simulation and detection of **T1136.001 — Local Account Creation** using **Atomic Red Team** on a Windows machine in a controlled lab environment.

Atomic Red Team executes the exact technique as mapped by MITRE ATT&CK, generating real Windows telemetry forwarded to Splunk for detection.

**Lab Platform:** Windows only  
**Domain:** `hettilava.local`

## Detection — Splunk SPL Queries

### Query 1 — Detect account creation (Event ID 4720)

```spl
index=wineventlog EventCode=4720
| table _time, host, Account_Name, SAM_Account_Name, Security_ID
| sort -_time
```

### Query 2 — Match Atomic Red Team default account names

```spl
index=wineventlog EventCode=4720
(Account_Name="atomic_test_user" OR Account_Name="*ANONYMOUS*" OR Account_Name="backdoor")
| table _time, host, Account_Name, SAM_Account_Name
| sort -_time
```

### Query 3 — Detect net.exe or New-LocalUser via Sysmon

```spl
index=sysmon EventCode=1
(Image="*\\net.exe" OR Image="*\\net1.exe"
 OR (Image="*\\powershell.exe" AND CommandLine="*New-LocalUser*"))
| table _time, host, User, Image, CommandLine
| sort -_time
```

### Query 4 — Detect account added to Administrators group (Event ID 4732)

```spl
index=wineventlog EventCode=4732 Group_Name="Administrators"
| table _time, host, Member_Security_ID, Group_Name, Subject_Account_Name
| sort -_time
```

### Query 5 — Full T1136.001 attack chain correlation

```spl
index=wineventlog OR index=sysmon
(EventCode=4720 OR EventCode=4732
 OR (EventCode=1 AND (Image="*net.exe" OR Image="*powershell.exe")))
| eval attack_phase=case(
    EventCode=4720, "1 - Account Created",
    EventCode=4732, "2 - Added to Group",
    EventCode=1,    "0 - Process Execution"
)
| table _time, host, attack_phase, Account_Name, CommandLine
| sort _time
```

## Analyst Notes

- Test #1 (`net user /add`) triggers both Security log EID 4720 and Sysmon EID 1 — easiest to detect
- Test #3 (`New-LocalUser`) bypasses net.exe-based detections — requires Sysmon + PowerShell script block logging
- The default account name `atomic_test_user` is an obvious IOC — real attackers use inconspicuous names like `svc_update` or `helpdesk2`
- Always run `-Cleanup` after testing to remove created accounts from the lab machine
- Ensure **Audit Account Management** is enabled in Group Policy; without it, EID 4720 will not generate
